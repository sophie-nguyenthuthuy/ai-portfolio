# Quiz — Section 16: CAREER & INTERVIEW

_8 câu · mỗi câu 1 đáp án đúng (✅) kèm giải thích._

---

**Câu 1.** Trong một repo portfolio, yếu tố nào nhà tuyển dụng coi trọng nhất?
- A. Code dài và phức tạp
- B. ✅ README rõ ràng + demo chạy được + có metric kết quả
- C. Số lượng commit
- D. Số ngôi sao
*Giải thích: người xem 30 giây phải hiểu dự án làm gì.*

**Câu 2.** Một bullet point CV tốt nên có cấu trúc nào?
- A. Chỉ tên công nghệ
- B. ✅ Hành động + công nghệ + kết quả số ("xây X bằng Y, đạt Z%")
- C. Càng dài càng tốt
- D. Chỉ mô tả nhiệm vụ chung chung
*Giải thích: luôn có con số đo lường được.*

**Câu 3.** ATS (Applicant Tracking System) là gì?
- A. Một mô hình AI
- B. ✅ Hệ thống tự động lọc CV trước khi tới tay người
- C. Một loss function
- D. Một database
*Giải thích: CV cần khớp từ khoá JD để qua máy.*

**Câu 4.** Trong phỏng vấn coding, điều gì quan trọng hơn cả việc ra đáp án đúng ngay?
- A. Code nhanh nhất
- B. ✅ Nói to suy nghĩ, làm rõ đề, phân tích cách tiếp cận
- C. Im lặng làm bài
- D. Dùng thư viện phức tạp
*Giải thích: nhà tuyển dụng đánh giá tư duy giải quyết vấn đề.*

**Câu 5.** Khung tư duy ML system design gồm các bước chính nào?
- A. Code → test → deploy
- B. ✅ Problem → Data → Model → Eval → Deploy
- C. Train → save → load
- D. Import → run → print
*Giải thích: có khung để không lạc khi trả lời; nhắc production = ghi điểm.*

**Câu 6.** Phương pháp STAR trong phỏng vấn hành vi là gì?
- A. Speed, Time, Accuracy, Result
- B. ✅ Situation, Task, Action, Result
- C. Start, Test, Apply, Review
- D. Skill, Talent, Ability, Role
*Giải thích: kể chuyện có cấu trúc, có kết quả.*

**Câu 7.** Khi đàm phán offer, nguyên tắc nào thường được khuyên?
- A. Luôn nói số lương mong muốn đầu tiên
- B. ✅ Tránh nói số đầu tiên, dựa vào giá trị mang lại & nghiên cứu thị trường
- C. Chấp nhận ngay offer đầu
- D. Chỉ quan tâm lương cứng
*Giải thích: portfolio 7 dự án là đòn bẩy; xét cả total compensation.*

**Câu 8.** Vì sao nên đóng góp open source (Hugging Face, scikit-learn, Airflow)?
- A. Để được trả tiền ngay
- B. ✅ Học từ code chất lượng cao, xây danh tiếng & mạng lưới, PR được merge là "chứng chỉ sống"
- C. Bắt buộc để tốt nghiệp
- D. Để có nhiều repo
*Giải thích: bắt đầu nhỏ (good first issue), chọn dự án đã dùng trong khoá.*
